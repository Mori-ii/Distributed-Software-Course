# 第五讲作业：事务与一致性

## 1. 场景说明

在商品秒杀系统中，订单服务和库存服务是两个独立的微服务，分别有各自数据库。本项目采用单体架构，通过 Kafka 消息队列实现分布式一致性。

## 2. Redis 库存预扣减

### 2.1 防超卖、限购策略

**Redis Lua 脚本原子扣减**：
```lua
-- seckill.lua
local voucherId = ARGV[1]
local userId = ARGV[2]
local orderId = ARGV[3]

local stockKey = 'seckill:stock:' .. voucherId
local orderKey = 'seckill:order:' .. voucherId

-- 1. 检查库存
if(tonumber(redis.call('get',stockKey)) <= 0) then
    return 1  -- 库存不足
end

-- 2. 检查是否重复购买（限购）
if(redis.call('sismember',orderKey,userId) == 1) then
    return 2  -- 不能重复下单
end

-- 3. 扣减库存（原子操作）
redis.call('incrby',stockKey,-1)

-- 4. 记录用户购买资格
redis.call('sadd',orderKey,userId)

return 0  -- 成功
```

### 2.2 库存预扣减流程
```
用户发起秒杀请求
      ↓
Lua脚本检查：库存是否充足？
      ↓
Lua脚本检查：用户是否已购买？（限购）
      ↓
Redis原子扣减库存
      ↓
记录用户购买资格
      ↓
发送Kafka消息
      ↓
返回订单ID给用户
```

### 2.3 防超卖原理
- Redis 单线程执行，保证原子性
- Lua 脚本将多个操作封装为原子操作
- 先检查再扣减，避免负数库存

## 3. 基于消息的一致性

### 3.1 Kafka 消息队列实现

**消息发送端**：
```java
@Override
public Result seckillVoucher(Long voucherId) {
    Long userId = UserHolder.getUser().getId();
    long orderId = snowflakeIdWorker.nextId();

    // Redis预扣减
    Long result = stringRedisTemplate.execute(SECKILL_SCRIPT, ...);
    if (result != 0) {
        return Result.fail(...);
    }

    // 构建订单消息
    VoucherOrder voucherOrder = new VoucherOrder();
    voucherOrder.setId(orderId);
    voucherOrder.setUserId(userId);
    voucherOrder.setVoucherId(voucherId);

    // 发送Kafka消息（异步）
    kafkaTemplate.send(KafkaConfig.SECKILL_ORDER_TOPIC, 
                       String.valueOf(userId), voucherOrder);

    return Result.ok(orderId);
}
```

**消息消费端**：
```java
@KafkaListener(topics = KafkaConfig.SECKILL_ORDER_TOPIC)
public void onSeckillOrder(VoucherOrder voucherOrder) {
    Long userId = voucherOrder.getUserId();
    
    // Redisson分布式锁
    RLock lock = redissonClient.getLock("lock:order:" + userId);
    boolean isLocked = lock.tryLock();
    if (!isLocked) {
        log.error("不允许重复下单，userId={}", userId);
        return;
    }
    try {
        proxy.createVoucherOrder(voucherOrder);
    } finally {
        lock.unlock();
    }
}
```

### 3.2 最终一致性保证
| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | Redis扣减库存 | 立即生效 |
| 2 | Kafka发送消息 | 异步 |
| 3 | 消费者落库 | 最终一致 |

## 4. 下单+库存扣减一致性

### 4.1 流程图
```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│   用户      │      │   Redis     │      │   Kafka     │
│  发起请求   │ ──▶  │  预扣库存    │ ──▶  │  发送消息   │
└─────────────┘      └─────────────┘      └──────┬──────┘
                                                 │
                                                 ▼
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│   数据库    │ ◀──  │  订单落库   │ ◀──  │   消费者    │
│  最终数据   │      │  扣减库存   │      │   接收消息   │
└─────────────┘      └─────────────┘      └─────────────┘
```

### 4.2 分布式锁保证
```java
@KafkaListener(topics = KafkaConfig.SECKILL_ORDER_TOPIC)
public void onSeckillOrder(VoucherOrder voucherOrder) {
    // 按用户ID加锁，保证同一用户串行处理
    RLock lock = redissonClient.getLock("lock:order:" + userId);
    boolean isLocked = lock.tryLock();
    if (!isLocked) {
        return; // 获取锁失败，跳过处理
    }
    try {
        proxy.createVoucherOrder(voucherOrder);
    } finally {
        lock.unlock();
    }
}
```

### 4.3 数据库层保证
```java
@Transactional
public void createVoucherOrder(VoucherOrder voucherOrder) {
    // 乐观锁：stock > 0 保证不超卖
    boolean success = iSeckillVoucherService.update()
            .setSql("stock = stock - 1")
            .eq("voucher_id", voucherId)
            .gt("stock", 0)
            .update();
    if (!success) {
        log.error("库存不足，voucherId={}", voucherId);
        return;
    }
    save(voucherOrder);
}
```

## 5. 订单支付+订单状态更新一致性

### 5.1 订单状态流转
```
CREATE(创建) → PAYING(支付中) → PAID(已支付) → USED(已使用)
    ↓                                              ↓
CANCELLED(已取消) ← WAIT_REFUND(退款中) ← REFUNDED(已退款)
```

### 5.2 支付回调一致性
```java
@PostMapping("/notify")
public Result payCallback(@RequestBody PayCallbackDTO dto) {
    // 1. 验证签名
    if (!verifySign(dto)) {
        return Result.fail("签名验证失败");
    }
    
    // 2. 查询订单
    VoucherOrder order = getById(dto.getOrderId());
    
    // 3. 状态机校验
    if (order.getStatus() != 1) { // 1=待支付
        return Result.fail("订单状态异常");
    }
    
    // 4. 更新状态
    order.setStatus(2); // 已支付
    order.setPayTime(LocalDateTime.now());
    updateById(order);
    
    return Result.ok();
}
```

## 6. TCC 事务（扩展）

TCC（Try-Confirm-Cancel）是一种两阶段提交协议：

### 6.1 Try 阶段
```java
@TwoPhaseBusiness
public void seckill(Long voucherId) {
    // 预留资源：冻结库存
    stockService.freezeStock(voucherId, 1);
}
```

### 6.2 Confirm/Cancel 阶段
```java
@Compensable(confirmMethod = "confirm", cancelMethod = "cancel")
public void freezeStock(Long voucherId, Integer count) {
    // 冻结库存
    stockMapper.decreaseAvailableStock(voucherId, count);
    stockMapper.increaseFrozenStock(voucherId, count);
}
```
