### 1.2 秒杀下单流程
```
用户请求 → Lua脚本检查库存+幂等 → Redis扣减库存 → Kafka发送消息 → 返回订单ID
                                    ↓
                         Kafka消费者接收消息
                                    ↓
                         Redisson分布式锁 → 订单落库
```

### 1.3 核心代码实现

**VoucherOrderServiceImpl.java** - 秒杀下单服务：
```java
@Service
public class VoucherOrderServiceImpl extends ServiceImpl<VoucherOrderMapper, VoucherOrder> 
        implements IVoucherOrderService {

    @Resource
    private KafkaTemplate<String, VoucherOrder> kafkaTemplate;

    /**
     * 秒杀下单入口
     * 1. Lua脚本原子判断库存+幂等
     * 2. Kafka异步发送订单消息
     * 3. 立即返回订单ID
     */
    @Override
    public Result seckillVoucher(Long voucherId) {
        Long userId = UserHolder.getUser().getId();
        long orderId = snowflakeIdWorker.nextId();

        // Lua脚本检查库存+幂等
        Long result = stringRedisTemplate.execute(
                SECKILL_SCRIPT,
                Collections.emptyList(),
                voucherId.toString(), userId.toString(), String.valueOf(orderId)
        );

        if (result != 0) {
            return Result.fail(result == 1 ? "库存不足" : "不能重复下单");
        }

        // 发送消息到Kafka
        VoucherOrder voucherOrder = new VoucherOrder();
        voucherOrder.setId(orderId);
        voucherOrder.setUserId(userId);
        voucherOrder.setVoucherId(voucherId);
        kafkaTemplate.send(KafkaConfig.SECKILL_ORDER_TOPIC, String.valueOf(userId), voucherOrder);

        return Result.ok(orderId);
    }

    /**
     * Kafka消费者：异步处理订单落库
     */
    @KafkaListener(topics = KafkaConfig.SECKILL_ORDER_TOPIC)
    public void onSeckillOrder(VoucherOrder voucherOrder) {
        // 使用Redisson分布式锁保证并发安全
        RLock lock = redissonClient.getLock("lock:order:" + userId);
        try {
            proxy.createVoucherOrder(voucherOrder);
        } finally {
            lock.unlock();
        }
    }
}
```

### 1.4 Redis Lua 脚本
```lua
-- 秒杀Lua脚本：原子检查库存+幂等+扣减
local voucherId = ARGV[1]
local userId = ARGV[2]
local orderId = ARGV[3]

local stockKey = 'seckill:stock:' .. voucherId
local orderKey = 'seckill:order:' .. voucherId

-- 检查库存
if(tonumber(redis.call('get',stockKey)) <= 0) then
    return 1  -- 库存不足
end

-- 检查是否重复下单
if(redis.call('sismember',orderKey,userId) == 1) then
    return 2  -- 重复下单
end

-- 扣减库存
redis.call('incrby',stockKey,-1)
-- 记录用户已下单
redis.call('sadd',orderKey,userId)

return 0
```

## 2. 雪花算法订单ID生成

### 2.1 SnowflakeIdWorker 实现
```java
@Component
public class SnowflakeIdWorker {
    // 结构: 1位符号位 | 41位时间戳 | 5位数据中心ID | 5位机器ID | 12位序列号
    
    private static final long EPOCH = 1640995200000L; // 2022-01-01
    
    private static final long DATACENTER_ID_BITS = 5L;
    private static final long MACHINE_ID_BITS = 5L;
    private static final long SEQUENCE_BITS = 12L;

    public synchronized long nextId() {
        long timestamp = currentTime();
        if (timestamp < lastTimestamp) {
            throw new RuntimeException("Clock moved backwards");
        }
        if (timestamp == lastTimestamp) {
            sequence = (sequence + 1) & MAX_SEQUENCE;
            if (sequence == 0) {
                timestamp = waitNextMillis(lastTimestamp);
            }
        } else {
            sequence = 0L;
        }
        lastTimestamp = timestamp;
        return ((timestamp - EPOCH) << TIMESTAMP_SHIFT)
                | (datacenterId << DATACENTER_ID_SHIFT)
                | (machineId << MACHINE_ID_SHIFT)
                | sequence;
    }
}
```

### 2.2 ID 结构说明
| 位段 | 长度 | 说明 |
|------|------|------|
| 符号位 | 1 | 固定为0 |
| 时间戳 | 41 | 毫秒级时间差 |
| 数据中心ID | 5 | 0-31 |
| 机器ID | 5 | 0-31 |
| 序列号 | 12 | 每毫秒内序号 0-4095 |

## 3. 幂等性保证

### 3.1 多重幂等保障

| 层级 | 实现方式 | 说明 |
|------|---------|------|
| Redis Lua | `SISMEMBER orderKey userId` | 防止同一用户同一商品重复下单 |
| 数据库 | 唯一索引 `UNIQUE(user_id, voucher_id)` | DB层兜底 |
| 代码逻辑 | `query().eq("user_id").eq("voucher_id").count()` | 查询校验 |

### 3.2 幂等实现代码
```java
@Transactional
public void createVoucherOrder(VoucherOrder voucherOrder) {
    Long userId = voucherOrder.getUserId();
    Long voucherId = voucherOrder.getVoucherId();

    // DB兜底幂等校验
    long count = query()
        .eq("user_id", userId)
        .eq("voucher_id", voucherId)
        .count();
    if (count > 0) {
        log.warn("重复下单被拦截，userId={}, voucherId={}", userId, voucherId);
        return;
    }

    // 扣减DB库存（乐观锁）
    boolean success = iSeckillVoucherService.update()
            .setSql("stock = stock - 1")
            .eq("voucher_id", voucherId)
            .gt("stock", 0)
            .update();
    
    save(voucherOrder);
}
```

## 5. 分库分表（选做）

当前项目未实现分库分表。后续可通过 ShardingSphere-JDBC 实现：

### 5.1 ShardingSphere 配置示例
```yaml
spring:
  shardingsphere:
    rules:
      sharding:
        tables:
          tb_voucher_order:
            actual-data-nodes: ds_${0..1}.tb_voucher_order_${0..15}
            table-strategy:
              standard:
                sharding-column: id
                sharding-algorithm-name: order_id_mod
            key-generate-strategy:
              column: id
              key-generator-name: snowflake
```

### 5.2 分片策略
- 按用户ID分库：保证同一用户的订单在同一库
- 按订单ID分表：保证订单ID全局唯一
