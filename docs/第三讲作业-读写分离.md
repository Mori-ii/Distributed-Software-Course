# 第三讲作业：读写分离

## 1. 读写分离概述

读写分离是一种常用的数据库性能优化策略，将读操作和写操作分离到不同的数据库服务器上执行，从而减轻主数据库的压力。

### 1.1 架构图
```
┌─────────────┐     ┌─────────────┐
│  主库       │     │  从库       │
│  (写操作)   │ ──▶ │  (读操作)   │
└─────────────┘     └─────────────┘
       │                   │
       └─────────┬─────────┘
                 │
         ┌───────▼───────┐
         │  应用层        │
         │ (读写路由)     │
         └───────────────┘
```

### 1.2 MySQL 主从复制配置

**主库配置 (my.cnf)**：
```ini
[mysqld]
server-id=1
log-bin=mysql-bin
binlog-do-db=seckill
```

**从库配置 (my.cnf)**：
```ini
[mysqld]
server-id=2
relay-log=relay-bin
read-only=1
```

### 1.3 Spring Boot 配置

项目中未实现读写分离配置，但可以通过以下方式实现：

**方式1：使用 ShardingSphere-JDBC**
```xml
<dependency>
    <groupId>org.apache.shardingsphere</groupId>
    <artifactId>shardingsphere-jdbc-core-spring-boot-starter</artifactId>
    <version>5.3.0</version>
</dependency>
```

**方式2：使用 Dynamic DataSource**
配置多个数据源，通过 AOP 切面动态切换。

### 1.4 测试验证

**测试读操作**：
```bash
# 观察请求是否路由到从库
mysql -h <slave-ip> -e "SELECT COUNT(*) FROM tb_user;"
```

**测试写操作**：
```bash
# 观察主从同步延迟
mysql -h <master-ip> -e "INSERT INTO tb_user VALUES(...)"
mysql -h <slave-ip> -e "SELECT * FROM tb_user"
```
